# 配方表助手 v2 实施计划（S1-S4 简化版）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 升级报告生成系统，对标行业最佳实践与权威文献（TTC 决策、Read-across 框架、SCCS/CIR/ECHA 风险评估等）。

**Architecture:** 4 个独立 Sprint，按"报告质量提升"链路排序。每个 Sprint 内部包含：数据层 (JSON) → 引擎层 (Python) → 报告层 (Word) → UI 层 (PySide) → 验证层 (生成测试报告)。

**Tech Stack:** Python 3.12, PySide6, SQLite, python-docx, json

**Spec 依据:** TTC 决策树（EFSA/WHO 2016）+ Read-across 框架（EFSA 2017）+ 化妆品安全技术规范（2015）+ SCCS Notes of Guidance 12th。

**明确不做：** 三项测试 UI（R&D 出真实报告）、用户角色、通知系统、报告模板自动选择、多产品联合暴露。

## Task 3: TTC 决策树（Sprint 3）

**Files:**
- Create: `userdata/ttc_crashdb.json` (50 常见原料的 Cramer 分类种子库)
- Modify: `safety_engine.py:571` (`assess_ingredient()` 改造)
- Modify: `report_generator.py` (`_build_comp_text()` 加 TTC 兜底分支)

**Steps:**

- [ ] **Step 3.1:** 创建 `userdata/ttc_crashdb.json`，结构：
  ```json
  {
    "version": "1.0",
    "thresholds_ug_per_kg_day": {
      "dna_reactive_mutagen": 0.0025,
      "cramer_i": 46.0,
      "cramer_ii_iii": 2.3
    },
    "ingredients": [
      {"name_zh": "甘油", "name_inci": "GLYCERIN", "cramer_class": "I"},
      {"name_zh": "乙醇", "name_inci": "ALCOHOL", "cramer_class": "I"},
      ...
    ],
    "default_class": "III"
  }
  ```
  种子至少 50 个常见原料（甘油、丁二醇、烟酰胺、透明质酸钠、水杨酸钠、苯氧乙醇、苯甲酸甲酯、PEG 类等），来源 CIR/SCCS Final Reports
- [ ] **Step 3.2:** 在 `safety_engine.py` 新增函数 `ttc_decision(ingredient_name: str, sed_mg_kg_day: float) -> dict`:
  - 查 ttc_crashdb.json 找 Cramer class
  - 找不到 → 默认 Cramer III
  - 算 `within_ttc = sed_ug_kg_day <= threshold`
  - 返回 `{'cramer_class', 'threshold_ug_kg_day', 'sed_ug_kg_day', 'within_ttc', 'method': 'TTC'}`
- [ ] **Step 3.3:** 改造 `assess_ingredient()` 函数（约 571 行），在 `usage.noael` 缺失时调用 `ttc_decision()`，结果塞进返回 dict 的 `toxicology` 字段
- [ ] **Step 3.4:** 改造 `report_generator.py` `_build_comp_text()` 函数的毒理学部分（约 662-668 行）：当 noael 缺失时改为输出 `基于 TTC 保守估计（Cramer X 类，阈值 Y μg/(kg·d)，本原料 SED=Z μg/(kg·d)，在/超出阈值）`
- [ ] **Step 3.5:** 验证：跑一个 NOAEL 缺失的原料评估（如 "甘油"），确认报告里出现"基于 TTC 保守估计（Cramer I 类，阈值 46 μg/(kg·d)）"
- [ ] **Step 3.6:** Commit: `feat(safety): TTC决策树（教材官方阈值 0.0025/46/2.3 μg/(kg·d)）`
