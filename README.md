# 备案配方表助手

一款专为化妆品企业设计的备案配方表管理工具，帮助您高效管理原料库、设计配方并生成符合监管要求的备案配方表。

## 功能特点

- **原料库管理**：支持批量导入、自动匹配INCI名称、供应商信息管理
- **配方设计**：可视化配方编辑，实时计算成分含量
- **目录同步**：自动抓取国家药监局已使用化妆品原料目录（目录I/II）
- **一键导出**：生成符合备案要求的Excel配方表

## 系统要求

- Windows 10/11 操作系统
- Python 3.8+
- 已安装 Edge 浏览器（用于目录同步功能）

### 2. 首次使用

首次运行时，应用会自动创建数据库和用户数据目录。建议先同步原料目录数据：

1. 点击「同步目录」标签页
2. 点击「同步目录I（已使用化妆品原料目录）」按钮
3. 等待爬取完成（约5-10分钟），数据存储在：`userdata/ingredient_catalog.json`
4. 点击「同步目录II（已注册化妆品原料目录）」按钮
5. 等待爬取完成（约5-10分钟），数据存储在：`userdata/ingredient_catalog_II.json`
6. 点击「原料库管理」标签页，点击「下载导入模板」按钮，编辑数据后；点击「导入Excel」按钮，系统会自动同步目录I和目录II的原料INCI名称数据。若INCI名称没有显示，点击「批量自动匹配INCI名称」按钮，系统会根据目录数据自动匹配原料INCI名称。
7. 点击「配方设计」标签页，输入单个原料名称或内部代码，开始设计配方设计，添加原料。或者点击「批量输入内部代码」框内容，用逗号、顿号或空格分隔按钮分割好内部代码，批量添加原料。

## 功能模块说明

### 一、配方设计

配方设计模块用于创建和编辑化妆品配方，生成符合备案要求的配方表。

#### 主要功能

1. **添加原料**
   - 在搜索框输入原料名称或内部代码
   - 系统自动显示匹配结果（从原料库中搜索）
   - 设置添加量（百分比）
   - 点击「添加」按钮加入配方

2. **配方表格**
   | 列名 | 说明 |
   |------|------|
   | 原料序号 | 原料在配方中的顺序编号 |
   | 标准中文名称 | 原料的中文名称 |
   | INCI名称 | 原料的INCI（国际化妆品原料命名）名称 |
   | 原料含量(%) | 该原料在配方中的添加百分比 |
   | 原料中成分含量(%) | 复合原料中各成分的占比 |
   | 实际成分含量(%) | 计算后的实际成分含量 |
   | 主要使用目的 | 原料的使用目的（如保湿剂、表面活性剂等） |
   | 原料报送码 | 原料报送时的唯一标识码 |
   | 备注 | 额外说明信息 |
   | 是否新原料 | 标识是否为新原料（0=否，1=是） |
   | 注册号/备案号 | 原料的注册或备案编号 |

3. **导出配方表**
   - 点击「一键导出备案配方表」按钮
   - 选择保存路径
   - 生成符合备案格式的Excel文件

#### 原料来源

配方设计中使用的原料来自于**原料库**。要在配方中使用原料，需先在「原料库管理」中添加原料，或通过Excel批量导入。

### 二、原料库管理

原料库管理模块用于管理企业的原料数据库。

#### 主要功能

1. **搜索原料**
   - 在搜索框输入关键词
   - 支持按原料名称、内部代码、INCI名称搜索

2. **添加原料**
   - 点击「添加原料」按钮
   - 填写原料信息：
     - 原料名称（必填）
     - 内部代码
     - 使用目的
     - 原料报送码
     - INCI名（留空自动匹配）
     - 成分组成（复合原料填写，如：水(70%)、甘油(30%)）

3. **编辑/删除原料**
   - 右键点击原料行
   - 选择「编辑」或「删除」

4. **批量操作**
   - **批量自动匹配INCI**：为未匹配INCI的原料自动匹配
   - **下载导入模板**：下载Excel导入模板
   - **从Excel导入**：批量导入原料数据

#### 导入模板格式

导入模板包含以下列：

| 列名 | 说明 | 必填 |
|------|------|------|
| 原料名称 | 原料的中文名称 | 是 |
| 成分组成 | 复合原料的成分构成（如：水(50%)、椰油酰甘氨酸钾(50%)） | 否 |
| 内部代码 | 企业内部编码 | 否 |
| 主要使用目的 | 原料的使用目的 | 否 |
| 原料报送码 | 原料报送码 | 否 |

#### 成分组成格式

复合原料的成分组成需遵循以下格式：
```
成分1(百分比%)、成分2(百分比%)、成分3(百分比%)
```

示例：
```
水(70%)、椰油酰甘氨酸钾(30%)
```

### 三、同步目录

同步目录模块用于从国家药监局网站抓取原料目录数据，为INCI自动匹配提供数据支持。

#### 主要功能

1. **同步目录I**
   - 抓取《已使用化妆品原料目录》I
   - 包含约8000+条原料记录
   - 数据存储在：`userdata/ingredient_catalog.json`

2. **同步目录II**
   - 抓取《已使用化妆品原料目录》II
   - 包含限用原料等特殊原料
   - 数据存储在：`userdata/ingredient_catalog_ii.json`

3. **目录信息展示**
   - 显示目录I/II的记录数量
   - 显示独特名称数量

#### 目录数据用途

同步的目录数据主要用于以下功能：
- **INCI自动匹配**：在添加原料或批量导入时，系统会自动根据原料中文名称匹配对应的INCI名称
- **成分INCI匹配**：复合原料的各成分也会自动匹配INCI名称

#### 使用流程

1. **同步目录数据**（首次使用或定期更新）
   ```
   同步目录I → 同步目录II → 完成
   ```

2. **添加原料时自动匹配**
   ```
   添加原料 → 输入中文名称 → 系统自动匹配INCI → 保存
   ```

3. **批量导入时自动匹配**
   ```
   下载导入模板 → 填写原料名称 → 导入 → 系统自动匹配INCI → 完成
   ```

#### 注意事项

- 同步目录功能需要网络连接
- 同步过程可能需要5-10分钟（取决于网络速度）
- 目录数据仅用于INCI匹配，不会直接添加到原料库中
- 如需在配方设计中使用目录中的原料，需先在原料库中添加

## 数据存储

应用数据存储在以下位置：

- **目录数据**：`userdata/ingredient_catalog.json`（目录I）

## 常见问题

### Q: INCI名称无法自动匹配？

A: 请确保已同步原料目录数据。如果仍然无法匹配，可能是原料名称与目录中的名称不一致，可手动填写INCI名称。

### Q: 导入Excel失败？

A: 请检查：
- Excel文件格式为.xlsx
- 列名与导入模板一致
- 成分组成格式正确

### Q: 导出的配方表格式不对？

A: 导出格式参考《已使用化妆品原料目录》标准格式，如果需要调整可联系开发人员定制。
