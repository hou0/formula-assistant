## **1. 项目背景与目标**

**背景**：

- 自2025年5月1日起，中国化妆品备案全面实施完整版安全评估报告制度，企业需提交符合《化妆品安全评估技术导则（2021年版）》的报告。
- 评估过程涉及多源数据比对、风险计算、报告撰写，人工操作耗时且易出错。
- 企业已有的生产配方转换为备案配方的小工具孤立运行，需要扩展为覆盖全流程的安评工具。
- 法规频繁更新，原料目录与评估数据动态变化，必须通过自动化手段持续跟踪合规性。

**产品目标**：\
打造一个从配方输入到安全评估报告输出的一站式智能工具，实现：

- 自动校验配方合规性（技术规范、已使用原料、国际评估结论等）。
- 精准识别风险物质并驱动补充评估。
- 按需执行多人群、多国别的暴露评估与风险计算。
- 生成符合中国（含完整版/基本结论）、欧盟（CPSR）等主要市场要求的安评报告。

## **2. 核心概念与术语**

**术语**

**定义**

**安全评估四步法**

危害识别 → 剂量-反应关系评估 → 暴露评估 → 风险特征描述

**SED**

**TTC**

## **4. 产品核心流程**

用户操作主流程：

1. **上传/输入备案配方** ── 由现有转换工具生成标准配方，或直接导入。
2. **补充产品信息** ── 使用部位、使用方法、适用人群、功效宣称、剂型、目标市场。
3. **启动合规检查** ── 系统逐原料执行三层比对（见 5.3），生成检查明细。
4. **风险物质排查** ── 自动识别配方中可能带入的风险物质，标注评估状态。
5. **暴露评估与安全评估** ── 根据人群、国别参数计算 SED/MoS，缺数据时触发 TTC/交叉参照。
6. **生成报告** ── 选择报告模板，系统根据报告类型和产品分类自动填充数据（含三大表），生成可编辑的 Word/PDF 初稿。
7. **人工复核与归档** ── 评估人员在线上补充、修订，签字确认，系统存档。

## **5. 功能需求详述**

### **5.1 配方管理模块**

**基础能力**（复用现有工具）：

- 导入生产配方，自动拆分复配原料，标准化为 INCI 名/中文名，输出备案配方表。
- 支持 Excel/CSV 导入导出，支持手动编辑调整。

**新增要求**：

- 配方表结构化存储，包含字段：原料序号、内部代码、INCI 标准中文名、INCI 英文名、CAS 号、使用目的、实际含量（%）、备注（风险物质标注）。
- 与安全评估引擎深度绑定，配方变更可触发重新评估。
- **多成分展开显示**：复配原料拆分后，支持多行展开显示各成分，主行显示原料基本信息，子行显示各成分详细信息（标准中文名称、INCI名称、成分含量、实际含量）。
- **子行合并功能**：对于多成分原料，自动合并原料名称、添加量、禁用、限用、使用参考、SED、综合结果等列，提升表格可读性。

**复配原料处理规则**：

- 复配原料中极其微量成分（如为保证原料质量添加的防腐剂、稳定剂等）也需要逐一说明并评估，否则易被驳回。系统应在复配原料拆分时自动提取这些成分并标记为“待评估”。

**香精评估分支**：

- 配方表仅填“香精”的：需按《技术导则》评估香精整体或证明符合 IFRA 标准。
- 同时列出具体香料组分的：需对每种组分单独评估。
- 儿童化妆品含香精的：需识别致敏成分。

管理所有动态、静态数据源，提供查询接口和更新机制。

**安全评估使用规则（已收录，待引擎集成）**：

两张使用信息表（`safety_usage_market` / `safety_usage_international`）顶部的使用说明已提取为脚本常量，包含4条核心规则：

1. **数据来源说明**：信息来源为注册备案有效化妆品，未进行系统安全性评价，使用者需自行承担安全责任。
2. **参考性质说明**：使用量数据为安全评估提供参考，需结合产品使用方法和作用部位评估适用性。
3. **驻留/淋洗互参规则**：相同作用部位的同一原料，若只有驻留类产品的原料使用量，淋洗类产品可参照驻留类使用。
4. **作用部位递进引用规则**：相同使用方法的同一原料，按全身→躯干→面部→口唇→眼部的顺序递进引用（A线），以及全身→躯干→手足→头部→头发→指(趾)甲（B线），并规定了跨线参照禁止规则（口唇、眼部不可参照B线部位）。

**版本管理功能**：

- 支持手动上传新版数据文件（CSV/Excel）或通过配置的 API/爬虫自动抓取官方公告。
- 自动执行版本对比：识别新增、删除、修改的原料条目；标记关键字段变更（名称、CAS 号、限制条件、历史使用量等）；生成差异报告，并推送预警给受影响的在评项目。
- 数据版本可追溯，可回滚至历史版本。
- **检验方法独立更新接口**：《技术规范》检验方法年度多次更新，架构层预留检验方法库的独立更新接口，与原料目录版本管理解耦。

### **5.3 合规性自动检查引擎**

**数据源引用优先级总则**：

《化妆品原料数据使用指南》规定的七种原料数据类型，在使用时须遵循以下强制优先级：

| **优先级** | **数据类型**         | **强制规则**                                          |
| ------- | ---------------- | ------------------------------------------------- |
| 1       | 《技术规范》限用/准用组分    | 必须优先满足。**必须优先满足**。若原料属于此类但不满足限制条件，**不得用其他证据类型替代** |
| 2       | 国际权威机构评估结论       | 需满足限制条件方可引用；不同机构结论不一致时，根据数据可靠性和相关性科学合理采用          |
| 3       | WHO/FAO等权威机构安全限量 | 需对相关资料进行分析，有充分数据支撑                                |
| 4       | 《已上市产品原料使用信息》    | 需结合使用方法和作用部位评估适用性                                 |
| 5       | 原料3年使用历史         | 需提供完整证明文件                                         |
| 6       | 安全食用历史           | 可豁免系统毒性，但仍需评估局部毒性                                 |
| 7       | 高分子聚合物（结构和性质稳定）  | 需满足相对分子质量要求                                       |
| 8       | 兜底               | 仅限不超过配方成分总个数10%且含量较低的非特殊功效原料                      |

**检查层次（必须按顺序）**：

**第一层：《化妆品安全技术规范》强制检查**

- 判定原料是否属于禁用组分 → 直接不合格。
- 判定是否限用组分：检查使用浓度、部位、pH、复配限制等。
- 判定是否准用防腐剂/防晒剂/着色剂/染发剂，校验合规性。
- 不满足则直接标记为不合规，**不进入后续证据类型检索**。

**第二层：已使用原料及上市使用量核查**

- 确认原料是否在《已使用目录》Ⅰ或Ⅱ中；不在且非已注册新原料则告警。
- 与《已上市产品原料使用信息》比对：对比配方浓度是否在历史使用量范围内，**必须考虑作用部位递进引用规则**（两条线，见 5.8.4.1）；超出历史用量或部位不适配时，发出“需进一步毒理学评估”提示。
- **跨线参照禁止规则**：口唇、眼部不可参照手足、头部、头发、指（趾）甲数据；头发、指（趾）甲不可参照口唇、眼部数据。

**第三层：国际权威评估结论与限制条件校验**

- 查询《国际化妆品安全评估数据索引》、CIR、SCCS 等结论。
- 若存在评估结论，**必须解析结论中的限制条件**，自动生成待办证明项。CIR 常见三类限制条件：

**限制条件类型**

**需提供的证明材料**

“当配方无刺激性时”

皮肤刺激性试验报告（急性/多次）；易接触眼部产品还需眼刺激性试验报告

“当配方无致敏性时”

皮肤变态反应试验报告

“当配方无刺激性和无致敏性时”

皮肤变态反应试验 + 皮肤刺激性试验；易接触眼部产品还需眼刺激性试验

- 未满足限制条件时，该原料不得引用该评估结论，系统自动降级推荐 TTC/交叉参照等方法。
- 多个机构结论冲突时，按可靠性/相关性标记，并提供人工选择入口。

**第四层：其他证据类型判定**

- 检查是否具备安全食用历史、3年使用历史、高分子聚合物判定等。
- 若仍无可用数据，且该原料不超过配方总个数10%且含量低，提示可开展终产品安全性测试（参照《化妆品注册和备案检验工作规范》）。
- 否则标记为“数据不足”，并建议开展毒理学试验或更换原料。

**引擎输出**：逐原料生成合规性状态（合格/有条件合格/不合格/存疑），附带详细引用依据及缺失证明清单。

### **5.5 安全评估引擎**

#### **5.5.1 暴露评估计算器**

- **人群分层**：婴幼儿（0-1岁 5kg、1-3岁 8kg、3-7岁 16kg）、儿童（7-10岁约24kg、11-14岁约38kg，参考日本AIST暴露因子手册）、青少年（12-18岁）、成人（男/女，默认60kg）、孕妇/哺乳期。
- **儿童化妆品暴露数据最新团标**：T/FDCA 13-2025《儿童化妆品典型品类暴露量》（2025年12月31日发布），膏霜类0-3岁1.06g/d、3-12岁1.53g/d；洗发沐浴类0-3岁2.30g/d、3-12岁3.54g/d。
- **参数来源**：中国境内参考《技术导则》附录参数及T/FDCA 13-2025；其他市场参考SCCS第12版（分4组儿童）、德国BfR修正数据、美国PCPC调查数据等。
- **计算公式**：SED = (日均使用量 × 驻留因子 × 成分浓度 × 经皮吸收率) / 体重。经皮吸收率默认100%，如有实验数据可调整。
- **儿童特殊计算**：对于牙膏等可能吞咽产品，需增加经口摄入分量。参考RIVM 2006报告：2.5岁儿童牙膏日均使用量1.72g（其中1.06g经口摄入）。欧盟SCCS指南中6岁以下儿童牙膏驻留因子40%。
- **多产品联合暴露**：对于儿童产品，需累加多产品联合暴露。以羟苯丁酯为例，SCCS最终意见指出所有化妆品中累计总暴露量不得超过**245 µg/kg bw/天**。

##### **5.5.1.1 日均使用量参考报告数据使用逻辑**

**数据来源优先级**：

| 优先级 | 数据来源    | 说明                              | 适用场景       |
| --- | ------- | ------------------------------- | ---------- |
| 1   | 企业自测数据  | 企业通过消费者调研或试验获得的实际使用量数据          | 有自测数据时优先使用 |
| 2   | 行业协会数据  | 如PCPC（美国）、JCIA（日本）发布的消费者使用量调查报告 | 无自测数据时使用   |
| 3   | 权威机构指南  | SCCS指南第12版、德国BfR修正数据            | 缺乏行业数据时使用  |
| 4   | 国家标准/团标 | T/FDCA 13-2025《儿童化妆品典型品类暴露量》    | 中国市场儿童产品专用 |
| 5   | 默认参数    | 《技术导则》附录推荐值                     | 兜底使用       |

**数据匹配规则**：

1. **产品类型匹配**：根据产品剂型（膏霜乳、水剂、气雾剂、贴膜、粉剂、蜡基）匹配对应的日均使用量数据。
2. **使用部位匹配**：根据作用部位（眼部、面部、口唇、头发、全身皮肤、手足、指趾甲、腋下）选择对应的暴露参数。
3. **人群匹配**：根据目标人群（婴幼儿、儿童、青少年、成人、孕妇/哺乳期）选择对应的体重和使用量数据。
4. **使用方式匹配**：区分驻留类和淋洗类产品，选择不同的驻留因子（驻留类默认1.0，淋洗类根据产品类型调整）。

**数据缺失处理策略**：

- **同类产品类比**：若无完全匹配数据，可参考同类产品的使用量数据，并在评估报告中注明数据来源和不确定性。
- **保守原则**：当存在多个数据源时，取最保守的值（即最大使用量）进行计算。
- **敏感性分析**：对关键参数进行敏感性分析，评估使用量变化对SED/MoS结果的影响。

**数据验证机制**：

- 系统自动校验使用量数据是否在合理范围内（如成人面部面霜日均使用量通常在0.5-2.0g之间）。
- 超出合理范围时，系统发出警告并提示用户确认或调整数据。
- 用户可手动输入自定义使用量数据，并记录数据来源说明。

#### **5.5.2 MoS 计算与风险判定**

- 获取 NOAEL（优先从数据库取，缺值时提示用户输入或启动替代评估）。
- 计算 MoS，低于 100 时高亮警示。
- 对儿童产品，适当调整可接受 MoS 阈值。

#### **5.5.3 替代评估方法**

##### **5.5.3.1 TTC 决策树（v2 新增：官方阈值落地）**

**官方阈值**（依据《化妆品安全评估基础与实践》p.136-142，中检院《TTC 方法应用技术指南》2024）：

| 分类 | 阈值 (μg/(kg·d)) | 适用条件 |
|---|---|---|
| DNA 反应性致突变物/致癌物 | 0.0025 | 含警示结构（黄曲霉毒素类、亚硝胺类、偶氮类等）禁用 TTC |
| Cramer I 类 | 46 | 低毒、简单结构、有安全食用/使用史 |
| Cramer II 类 | 2.3 | 中等毒性，结构中等复杂 |
| Cramer III 类 | 2.3 | 高毒、复杂结构、警示基团 |

> **注**：行业流传的"9/1.5 μg/day"为旧版分类，**官方已合并为 2.3**。

**评估流程**（7 步）：
1. 判断原料是否符合 TTC 适用范围（结构明确 + 含量低 + 缺系统毒理学数据 + 无警示结构）
2. 计算 SED (μg/(kg·d))
3. 判断是否为 DNA 反应性致突变物/致癌物（试验数据 + 2 类互补 (Q)SAR 工具）
4. 若是 → 与 0.0025 μg/(kg·d) 比较
5. 若否 → 查 Cramer 分类（默认 Class III 保守）
6. 与对应阈值比较
7. 输出结论：可接受 / 需进一步评估

**Cramer 分类工具**：Toxtree（JRC）+ QSAR Toolbox（OECD）。v2 实现：先用关键词表自动分类（保守默认 III），后续可接 Toxtree API。

##### **5.5.3.2 交叉参照 Read-across（v2 新增：7 步框架）**

**官方依据**：中检院《交叉参照（Read-across）方法应用技术指南》2024.4.30 发布。

**适用范围**：结构明确、缺乏系统毒理学数据的**非功效成分或风险物质**。**不适用**于防腐、防晒、着色、染发、祛斑美白类原料。

**7 步评估框架**：
1. 目标化学物质信息整理（结构 + 理化 + 代谢 + 反应活性）
2. 识别与确定潜在类似物（OECD Toolbox / EPA CompTox GenRA / AIM / Toxmatch）
3. 收集源化学物质数据（结构相似性 + 理化性质 + 毒理数据）
4. 构建数据矩阵（源-目标逐项对比）
5. 验证合理性 + 不确定性分析
6. 填补数据缺口
7. 得出结论并形成报告

**v2 落地形态**：评估时支持指定"类似物"参数，报告里自动生成"由于无本品毒理数据，参考类似物 X 的 NOAEL=Y..."段落。类比物映射表留待扩展。

##### **5.5.3.3 NGRA 基础（占位）**

- 允许上传 in silico（(Q)SAR、Toxtree）、in vitro（DPRA、KeratinoSens、h-CLAT）测试结果并纳入评估证据链
- 暂未实现可视化 UI，数据通过 `usage.extra_evidence` 字段传入

#### **5.5.4 多市场参数集成**

- 为每个支持的市场定义一组暴露参数和禁用/限用清单。
- 评估时，根据目标市场自动切换参数集和法规库，重新计算 SED、MoS，标记各国特定关注项。

#### **5.5.5 毒理学数据管理子模块（v2 升级）**

##### **5.5.5.1 官方 NOAEL/LOAEL 公式（v2 新增）**

- MoS = NOAEL / SED，**阈值 = 100**
- 28 天重复剂量毒性试验 NOAEL 需 × **3 倍不确定因子**（即 28d NOAEL ÷ 3 用于 MoS 计算）
- 无 NOAEL / BMD 时用 LOAEL 计算 MoS，**同样 × 3 倍不确定因子**
- 毒理学数据质量有缺陷时 MoS 阈值需适当提高（>100）

##### **5.5.5.2 毒理学试验方法规范**

系统内置《化妆品安全技术规范》中收录的毒理学试验方法目录。

**动物替代方法管理（IATA 整合策略）**：\
根据《皮肤致敏性整合测试与评估策略应用技术指南》，我国现阶段采用“3选2试验”策略：基于AOP框架整合DPRA、KeratinoSens/LuSens、h-CLAT等方法。

使用替代方法的优先级判定：

1. 优先使用已收录于《化妆品安全技术规范》的替代方法。
2. 使用未收录方法时需说明方法适用性、检测的关键事件、结果判定及不确定性分析。
3. 所用方法须为国际权威替代方法验证机构已收录的方法。

#### **5.8.5 特殊人群差异化处理**

##### **5.8.5.1 儿童产品（含婴幼儿）专项评估**

1. **配方设计原则检查**：儿童化妆品应遵循“安全优先、功效必需、配方极简”三原则。系统可检查：是否使用了监测期内新原料（应禁止）、是否使用了基因/纳米技术原料（应禁止）、是否含有祛斑美白/祛痘等成人功效原料（应禁止）。
2. **暴露参数切换**：体重不采用成人默认60kg，改用分龄体重；日均使用量优先采用T/FDCA 13-2025团标数据；6岁以下牙膏驻留因子40%。
3. **毒理学试验要求**（不可豁免）：眼刺激性应为无刺激性或微刺激性；皮肤刺激性应为无刺激性；皮肤变态反应应为无致敏性；皮肤光毒性应为无光毒性。
4. **微生物与理化指标**：儿童化妆品菌落总数≤500 CFU/g（或CFU/mL）；驻留类pH 4.5～7.5，淋洗类pH 4.5～8.5。
5. **经口暴露分量**（婴幼儿0-3岁）：对于手部产品、唇部产品等有吸吮/舔食可能的，自动增加经口暴露计算。

##### **5.8.5.2 孕妇/哺乳期专项提示**

- 自动检查配方中是否含有需关注的成分（如视黄醇及其衍生物、高浓度水杨酸、部分化学防晒剂等）。
- 提示需额外考虑发育毒性/生殖毒性终点。

### **5.13 特殊产品类型专项评估模块**

#### **5.13.1 气雾剂型产品**

**推进剂分离评估**：将推进剂与其他原料分开评估，其他原料的评估浓度按扣除推进剂后配方中各组分浓度计算。

**吸入毒性评估**：对存在吸入毒性风险的推进剂，还需评估其吸入毒性。

**系统行为**：检测到推进剂时自动切换为“气雾剂模式”；触发吸入暴露评估提示；按扣除推进剂后的浓度重新计算SED。

#### **5.13.2 贴、膜类产品**

**载体材料评估**：需注明主要载体材料的材质组成，提供来源、制备工艺、质量控制指标等资料。面膜材质发生变化时需对载体材料的稳定性重新评估。

**系统行为**：剂型为贴膜类时自动增加载体材料管理字段；载体材料与内容物配方分开管理。

#### **5.13.3 纳米原料**

产品配方中含有纳米原料的，应明确纯度、晶型、初始粒径分布、表面涂层物质等信息的原料质量规格，并说明毒理学试验方法是否适用于纳米原料检测。

#### **5.13.4 防晒类产品**

SCCS第12版指南中防晒产品有独立暴露参数表，通常使用量高于普通驻留类产品，且需考虑户外补涂频次。工具应使用防晒专用暴露参数而非通用驻留类参数。

#### **5.13.5 新原料安全监测**

经注册/备案的新原料投入使用后3年内，每年报告使用和安全情况。含监测期新原料的产品应提交毒理学试验报告（不得自检），且自动升级为第一类化妆品。

#### **5.13.6 两剂或两剂以上配合使用的产品**

应根据产品的使用方法对混合后的原料含量进行评估。

### **6.4 风险物质规则库**

**实际存储形式**：JSON文件 `userdata/risk_substances.json`

**文件结构**：

```json
{
  "version": "1.1",
  "last_updated": "2026-06-02",
  "reference": "《化妆品风险物质识别与评估技术指导原则》",
  "general_risk_substances": [...],  // 通用风险物质（13种）
  "child_risk_substances": [...]     // 儿童化妆品特殊风险物质（3种）
}
```

**每条规则的字段**：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `name` | string | 风险物质名称（如"二𫫇烷"） |
| `trigger_keywords` | array | 触发关键词列表（原料名称包含任一关键词时触发） |
| `trigger_all` | boolean | 是否所有产品都需要检测（true=全部，false=按关键词匹配） |
| `limit` | string | 限值要求（如"≤10mg/kg"） |
| `reference` | string | 参考依据（法规文件或公告） |
| `description` | string | 表3备注内容，用于报告生成 |

**示例规则**：

```json
{
  "name": "二𫫇烷",
  "trigger_keywords": ["PEG", "聚乙二醇", "乙氧基", "聚山梨醇酯", "脂肪醇聚醚", "泊洛沙姆"],
  "trigger_all": false,
  "limit": "≤10mg/kg（2027年1月1日起实施）",
  "reference": "国家药监局公告（2026年）",
  "description": "化妆品终产品中二𫫇烷的残留浓度应符合《化妆品安全技术规范》（2015年版）第一章<概述>中表2\"化妆品中有害物质限值\"的要求..."
}
```

## **7. 非功能需求**

- **性能**：单配方（30原料内）合规检查响应时间<10秒；完整报告生成<30秒。
- **可扩展性**：支持通过配置文件或插件新增国家市场支持，无需修改核心代码。
- **数据安全**：配方信息加密存储，支持私有化部署。
- **易用性**：提供清晰的原料状态标识（绿灯/黄灯/红灯），检查结果可直接导出Excel供人工复核。
- **法规响应**：规则引擎不应硬编码，可通过JSON/YAML配置实现快速调整。

## **8. 与现有配方转换工具的集成**

- 保留现有Python配方转换脚本，将其封装为独立微服务或库。
- 转换完成后，自动将标准配方推送至安评系统，触发新建评估项目。
- 配方变更时，系统记录版本差异，并提示重新评估。
